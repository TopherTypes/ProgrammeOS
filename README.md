# Programme Management App

A desktop-first web application designed for programme and project
leadership, focused on attention management, governance tracking, and
structured communication.

The system helps programme managers coordinate projects, people,
meetings, decisions, and actions while ensuring that important updates
are communicated to the right stakeholders.

## Core Philosophy

Most project management tools optimise for task tracking. This tool
optimises for leadership attention.

Key questions the system helps answer:

-   Which projects need attention right now?
-   Which stakeholders still need to be informed about recent decisions?
-   What risks and issues are emerging across the programme?
-   Which actions are overdue or blocking progress?
-   When did I last speak to key people?

## Core Features

### Dashboard

An attention-driven control surface showing: - Projects requiring
review - People requiring meetings - Overdue actions - Pending
communication items - Meeting preparation workspace

Each panel links to the full entity view.

### Project Workspace

Projects behave as operational workspaces containing: - Overview -
RAID - Updates - Meetings - People (roles) - Actions - Decisions

Tracked fields include start date, target date, review cadence, last
review date, and health status.

### Communication Tracking

Updates, decisions, and actions track who needs to be informed and when
they were informed.

### RAID Tracking

A single RAID table filtered by: - Risk - Action - Issue - Decision

### Meetings as Workspaces

Meetings support: - agenda preparation - communication tracking -
decision tracking - action capture

### Progressive CRUD

Creation can occur: - globally - contextually within entities
